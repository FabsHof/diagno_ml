# Integration Tests Failing: PostgreSQL role 'root' does not exist

## Problem
The integration tests in the CI pipeline are failing with a PostgreSQL connection error.

**Failing Job:** https://github.com/FabsHof/diagno_ml/actions/runs/21598980347/job/62239415874

## Error Details
```
FATAL:  role "root" does not exist
```
The job exits with code 5 and reports "no tests ran in 0.04s".

## Root Cause
The integration tests or database setup are attempting to connect to PostgreSQL using the "root" role, but the workflow configuration specifies a different user.

According to `.github/workflows/ci.yml`, the postgres service is configured with:
- POSTGRES_USER: test_user
- POSTGRES_PASSWORD: test_password
- POSTGRES_DB: test_db

## Solution
1. Update any database connection code that references "root" user to use "test_user" instead
2. Ensure all database connections use the correct URL format:
   ```
   postgresql://test_user:test_password@localhost:5432/test_db
   ```
3. Check test setup files, fixtures, and any database configuration that might override the `DATABASE_URL` environment variable

## Files to Check
- Integration test files in `tests/integration/`
- Database configuration/setup files
- Test fixtures or conftest.py that initialize database connections

## Expected Behavior
Integration tests should connect to PostgreSQL using the credentials defined in the workflow file and run successfully.
